General Notes

  • Inline element don't have line break, respect margin and padding horizontally but not vertically, can't have width and height set. Block elements create line break, respect margin and padding both dimensions, can have width and height set. Inline-block don't create line break, respect margin and padding both dimensions, can have width and height set. Converting an elemnent to display:flex/inline-flex causes element to behave similar to block/inline-block elements.
  • Block elements like div, ul, li take as much width possible for parent element but not height. Setting element as flex or grid, allows to set explicitely width and height of block child elements.
  • Flex direction of row makes the child elements to stretch to fill the height and with column to fill the elements width.
  • Avoid using ul elements inside ul elements and div elements being exact children of ul elements.
  • The correct order for anchor elements to cascade properly is ":link", ":visited", ":hover", and ":active".
  • Height and width percentages are proportional to the heigh and width of the parent container. If want to use percentages for margin/padding, this will depend only on the width of the parent container. Also, for both cases, the width/height of the parent must be set first, if not weird things will happen.
  • Important to use media queries to consider the different sizes that exist between devices. Should first look at the breakpoints of the site (by resizing the webiste and searching where it stops to look good). A normal breakpoint is the 'max-width=480px' which is the normal size of a phone.
  • A media consists of "@media only screen and (media feature)". 'only screen' details that styles will only apply to screens, this value can also be printers, among others. Then the features are the conditions that decide if the CSS should be applied, and they are separated by and 'and' or a ','(comma) that represents an 'or'.
  • There are two scenarios that will need to scale image proportionally.
    • First case is when you have an image inside the container in the html, to scale it you need to: solution. Here the overflow avoids the image to be displayed outside the container. The max-width makes image span the entire container and auto height to keep the image consistent and display:block to be on their own and not together with other elements (inline-block, the default state)
    • Second scenario is to add a background image to an element, to scale it you need to: solution. Here if the container is too big the image will not repeat itself (by default images repeat). It will also be always centered. The cover will make the image cover the entire element while keeping the proportions, and it will hide the image if it overflows.

Grid Customization

Property Possible Values Description
display grid - inline-grid To convert an item into a container and declare if will ocuppy the whole block of view or will be inline with other grids.
justify-self or align self start, end, center, stretch This goes for individual items and override the properties justify-items and align items.
justify-items start, end, center, stretch This positions the grid items within the columns, horizontally. Start/end: aligns items at the start/end. center: center, stretch: stretch the items to fill the grid area. without justify-items elements span entire width. center will make them be wide as necessary but no more.
align-items start, end, center, stretch This positions the grid items within the rows, vertically. Start/end: aligns items at the start/end. center: center, stretch: stretch the items to fill height of row. without align-items elements span entire height. center will make them be tall as necessary but no more.
justify-content start, end, center, stretch, space-around, space-between, space-evenly This aligns the columns along the container. start/end/center: aligns the columnsin this container. stretch: stretch the containers horizontally across container. space-around: equal amount of space left and right of everything so causes double space between elements. space-between: includes equal amount space but no space at start/end. space-evenly: space evenly between items.
align-content start, end, center, stretch, space-around, space-between, space-evenly This aligns the rows along the container. start/end/center: aligns the rows inside container. stretch: stretch the containers vertically across container. space-around: equal amount of space left and right of everything so causes double space between elements. space-between: includes equal amount space but no space at start/end. space-evenly: space evenly between items. E.g. the center aligns the rows to be vertically centered inside the container.
grid-template-areas add names as layout of the rows/columns. After giving size to rows and columns, can name each space like a name. To assign other elements that name with grid-area so they are in that area.
grid-area grid-row-start / grid- column-start /grid-row-end / grid-column-end Abbreviation for grid-row and grid-column.
grid-row or grid-column grid where item starts + '/' + grid where item ends. If a container has 3 rows and want an item to be in the first two rows, them the start is in 1 and the end is in 3th grid. Because 3 rows have 4 grids. The same applies for columns. Instead of bein specific, can use 'span 2' at the end to detail that an items should span 2 columns/rows.
gap gap between the row and then gap between columns. Of the availabe width/height, the gap will take the value and reduce the available space for the items.
grid-auto-rows or grid-auto-columns px, %, fr, repeat() If you don't how many element there will be, and you add X new elements then the grid will add automatically a row/columns. You use this property to define the size of the row/column.
grid-auto-flow row, column, dense Specify if there are new elements added, if they should be added to a new row or a column. Dense will try to put holes at the start and fill it with items.
grid-template defining the size of each row, then a '/' and then size of eac column. Define the size of each row and then the size of each column. Use a '/' to separate the different sizes. Can use the 'fr' unit to separate the size in fractions of the grid and prevent overflowing. Can also use repeat()to write less, where repeat(3,1fr) = 1fr 1fr 1fr. If container have variable width (i.e no width specified) then can use minmax() in a row/column to limit the size.
grid-template-columns defining the size of each column Each value will be a single column's width. If use percentage, it will depend on the parent's width.
grid-template-rows defining the size of each row Each value will be a single row's width. if use percentage, it will depend on the parent's height.

Flexbox Customization

Property Possible Values Description
display 'flex' - 'inline-flex' For cointainer. If display is set flex, the element will be a flex container but will occupy the entire block. If it is set as inline-flex it will cause two flex containers to be in the same line. Both cases cause children to be inline.
justify-content flex-start, flex-end, center, space-around, space-between For container. flex-start: aligns children start (horizontally for rows and vertically for columns. main axis affected, which is the one the direction points). flex-end: aligns children to end. center: center the children. space-around: have 1unit space before and 2 units space between children elements. space-between: have 0units before and all space between children.
align-items flex-start, flex-end, center, baseline, stretch For cointainer. flex-start: position items at top of element (vertically for row and horizontally for columns. cross axis affected, which is the perpendicular direction where it is pointing). flex-end: positione children at the bottom of element. center: position children center of element horizontally. basline: align bottom of all children. stretch: children with min height or no height, will stretch and fill container.
flex 3 possible values Can insert flex-grow, flex-shrink and flex-basis in one line inside the flex.
flex-grow units For children. Specify how much the children will grow when the container is growing. This means this will keep more of the extra space than other elements.
flex-shrink units For children. Specify how much the children will shrink when the container is shrinking.
flex-basis units For children. Specify the width of an element before it streches or shrinks.
align-content flex-start, flex-end, center, space-between, space-around, stretch Similar to align-items, but this one tells how couple of rows or more will be aligned. flex-start aligns all rows in the top, flex-end aligns rows bottom, center aligns at cetner, space-between aligns rows evenly with no space above top nor below bottom, space-around space evenly above and below each element (this causes to two items togther to be space double)
flex-flow flex-direction , flex-wrap Define flex-direction and flex-wrap in same line
flex-wrap wrap, nowrap, wrap-reverse For container. wrap: If there is no space in a single row, it will create a new row in next line (below) when resizing the page. wrap-reverse: wrap but in reverse (a line above), nowrap -
flex-direction row, column, row-reverse, column-reverse For cointainer. Tells how the children will be display.

General Customization

Property Possible Values Description
box-sizing border-box Can be applied at the start of the website to all elements (*). Makes size of an element more predicable by resizing the content to have a whole number and not to be modified/added by margin-padding
position static, relative, absolute, fixed, sticky Static: element positioned according to normal flow of document, no effect. Relative: Postioned according to normal flow of element but can then offset (moved) based on the original position. Absolute: Ignores all elements and position relative to the ancestor. Fixed: element will appear in the same place every time, even if you scroll. Sticky: Similar to fixed, but the element will disappear when the ancestor of the element finishes and is not in current page.
content Any string Can use the '::after or ::before, etc' to select an element and then add the content to the element.
font-style Can be all the possible fonts that exist Can use a font from google and apply a fallback if the font can't be rendered like sans or sans-serif.
font-weight Can be a number or 'bold' Details the thickness of a word.
margin Four possible values that detail the margin top, right, bottom. left Detail the space between the selector and the other elements.
padding Four possibles values: top-right-bottom-left Detail the space between the content contained on the selector and the border of the selector.
background -image 'url(url...)' Details a background-image that will be contained on the selector.
max-width or min-width number, percentage, rem, em The website adapts when the page increase or decreases. This property stops the content from adapting when reaching a max/min width..
border-radius number values Details how rounded the corners of a border will be. 50px makes a circle.
display inline-block or inline, or block Details if certain elements can be together in the same line or will span the whole width of the page. Inline works to put together the text (dont have margin or width/height). Inline-block is used when want to put blocks together, where want to modify the width/height/margin, etc to style the block.
list-style 'none' or 'url()' None removes the default option for bullet points of unordered lists. Can also add a url to customize it.
list-style-position 'inside' This applies to ul or ol. It details that the bullet points should be inside the parent container of the ul/ol.
opacity 0-1 1 is full opaque, full resolution. 0 value is full transparent, can't see it.
text decoration 'none', 'underline', etc Can be applied to texts and also to remove the underline of the anchor element.
text-transform 'uppercase', 'lowercase' transforms the text to all uppercase or lower case.
text align 'center', 'right', 'left' Align the content based on the parent element of the selector.
text transform 'uppercase', 'lowercase' transforms the text to all uppercase or lower case.
letter-spacing, word-spacing, line-height Number Modifies the space between the words. It accepts percentages, ems, rems, and pixels.
border widthBorder, styleLine (solid/ dashed) or color Need to set table, th, and td to have a border to show the gridlines in the browser.

Table Customization

Property Possible Values Description
border-collapse 'collapse'/'separate' If set to 'collapse' then cells inside a table will share borders. If set to 'separate', then each cell will have different borders. This only applies when targerting the table directly.
border-spacing numer If this is 0, it will appear as if there is no separation between cells.
